На основе материалов Части 7 курса, ниже представлен готовый программный код для реализации системы сессий (логин/логаут) через SessionsHelper, а также код для интеграции аватаров через сервис Gravatar.
1. SessionsHelper (app/helpers/sessions_helper.rb)
Этот хелпер содержит основную логику управления состоянием входа пользователя.
module SessionsHelper
  # Выполняет вход пользователя
  def sign_in(user)
    remember_token = User.new_remember_token
    # Сохранение токена в постоянные куки браузера
    cookies.permanent[:remember_token] = remember_token
    # Обновление зашифрованного токена в базе данных
    user.update_attribute(:remember_token, User.encrypt(remember_token))
    self.current_user = user
  end


  # Сеттер для текущего пользователя
  def current_user=(user)
    @current_user = user
  end


  # Геттер для извлечения текущего пользователя по токену из куки
  def current_user
    remember_token = User.encrypt(cookies[:remember_token])
    # Использование мемоизации: поиск в БД только при первом вызове
    @current_user ||= User.find_by(remember_token: remember_token)
  end


  # Проверка, вошел ли пользователь в систему
  def signed_in?
    !current_user.nil?
  end


  # Выход из системы
  def sign_out
    unless current_user.blank?
      # Смена токена в БД для безопасности
      current_user.update_attribute(:remember_token, User.encrypt(User.new_remember_token))
      # Удаление куки
      cookies.delete(:remember_token)
      self.current_user = nil
    end
  end
end

3. Интеграция Gravatar (app/helpers/users_helper.rb)
Функция для автоматического получения аватара пользователя по его email.
module UsersHelper
  # Возвращает изображение Gravatar для данного пользователя
  def gravatar_for(user)
    # Создание MD5 хэша из email (в нижнем регистре)
    gravatar_id = Digest::MD5::hexdigest(user.email.downcase)
    gravatar_url = "https://secure.gravatar.com/avatar/#{gravatar_id}"
    image_tag(gravatar_url, alt: user.name, class: "gravatar")
  end
end

4. Вспомогательные методы в модели User (app/models/user.rb)
Для работы токенов сессии в модели должны быть определены методы генерации и шифрования:
class User < ApplicationRecord
  # ... (валидации и callbacks) ...


  # Генерация случайного токена (base64)
  def User.new_remember_token
    SecureRandom.urlsafe_base64
  end


  # Шифрование токена алгоритмом SHA1
  def User.encrypt(token)
    Digest::SHA1.hexdigest(token.to_s)
  end


  private


  # Callback для создания токена при регистрации
  def create_remember_token
    self.remember_token = User.encrypt(User.new_remember_token)
  end
end

5. SessionsController (app/controllers/sessions_controller.rb)
Контроллер обрабатывает запросы на вход и выход.
class SessionsController < ApplicationController
  def new
    # Отображает страницу входа
  end


  def create
    unless params[:session].blank?
      user = User.find_by(email: params[:session][:email].downcase)
      # Проверка пользователя и аутентификация пароля (bcrypt)
      if user && user.authenticate(params[:session][:password])
        sign_in user
        redirect_to work_url # Переход в рабочую область
      else
        flash.now[:alert] = 'Login failed. Invalid email/password combination.'
        render 'new'
      end
    end
  end


  def destroy
    sign_out
    redirect_to root_url
  end
end
